Step 1

1 Copy Files to XAMPP

1
Extract the zip — you'll get a folder called trueocc
2
Replace your existing folder — copy everything from the extracted trueocc/ into C:\xampp\htdocs\trueocc\ and overwrite all files
3
Start XAMPP — open XAMPP Control Panel → Start both Apache and MySQL
4
Open Chrome (must be Chrome or Edge — Firefox has limited PWA support)
Step 2

2 Verify PWA is Working in Chrome

1
Open Chrome and go to: http://localhost/trueocc/
2
Press F12 → click the Application tab → look at the left panel
3
Click Service Workers — you should see sw.js with status ✅ activated and running
4
Click Manifest — you should see app name True Occupation, icons, and theme color #0A66C2
5
Click Cache Storage — expand it to see trueocc-v1.0.0-static with all your cached files listed
6
Look at the bottom-right of the screen — you should see a blue 📲 Install App button appear after 3 seconds
Step 3

3 Install on Desktop (Chrome / Edge)

Method A — Using the Install Button

1
Visit http://localhost/trueocc/ — wait 3 seconds
2
Click the blue 📲 Install App button in the bottom-right corner
3
Click Install in the Chrome popup — TrueOcc opens as a standalone window!

Method B — Using Chrome Address Bar

1
Visit http://localhost/trueocc/
2
Look for the ⊕ install icon in the Chrome address bar (right side)
3
Click it → click Install
Step 4

4 Install on Android Phone

⚠️ Your phone and PC must be on the same WiFi network. You need your PC's local IP address.
1
On your PC, open Command Prompt and type ipconfig — find your IPv4 Address (e.g. 192.168.1.5)
2
On your Android phone, open Chrome and go to: http://192.168.1.5/trueocc/ (replace with your actual IP)
3
Chrome will show a banner at the bottom: "Add TrueOcc to Home Screen" — tap it
4
Or tap the 3-dot menu → Add to Home Screen → Add
5
The TrueOcc icon appears on your home screen — tap to launch it in full-screen app mode!
✅ The app will open without browser bars — it looks and feels like a native Android app!
Step 5

5 Test Offline Mode

1
Open Chrome DevTools → F12 → Network tab
2
Change the throttling dropdown from "No throttling" to "Offline"
3
Refresh the page — it should still load from cache! The red "📡 You are offline" bar appears at the top
4
Navigate between pages — they all load from cache
5
API calls (login, jobs) show an offline error message — this is correct behavior (PHP needs server)
6
Set throttling back to "No throttling" — the green "✅ Back online!" bar appears
Step 6

6 Run Lighthouse PWA Audit

1
Open Chrome DevTools → F12 → click Lighthouse tab
2
Uncheck everything except Progressive Web App — click Analyze page load
3
Wait ~30 seconds — you'll get a PWA score and checklist of what's passing
4
Expected results: ✅ Installable · ✅ Service Worker · ✅ Offline page · ✅ Manifest
⚠️ HTTPS is required for a perfect 100% PWA score. On localhost this is waived — you'll still get the install prompt. For 100% score, deploy to a server with HTTPS.

Troubleshooting

⚡ Common Issues & Fixes

Install button not showing

Open DevTools → Application → Manifest — look for errors. Most common: manifest not found or icons not loading. Make sure all files were copied correctly and Apache is running.

Service Worker shows "redundant"

In DevTools → Application → Service Workers → click Unregister → hard refresh with Ctrl+Shift+R. The new SW will register fresh.

Old cached pages showing after update

Change APP_VERSION in sw.js from trueocc-v1.0.0 to trueocc-v1.0.1. Old caches are automatically deleted on next SW activation.

Android: "Add to Home Screen" not appearing

Must use Chrome (not Samsung browser). Also make sure you've visited the site for at least 30 seconds and interacted with it. Chrome requires user engagement before showing the prompt.
